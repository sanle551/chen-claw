# 加密货币代币深度分析与多维度投资备忘录

## 核心设计原则

- **链上优先**：链上数据不会撒谎，社交媒体会
- **三维度交叉验证**：链上数据(Bitget) + 新闻情绪(6551) + 社交热度(Twitter)
- **安全审计前置**：任何 swap 前先过安全扫描
- **可执行决策**：明确入场价格、止损线、目标位、仓位管理
- **反 rug pull 设计**：内置常见骗局识别模式

## 执行总流程

```
第零步：Token 基础信息获取（合约、链、市值）
第一步：安全审计（合约风险扫描）
第二步：16大分析模块（A-P）
第三步：6大投资哲学视角审视
第四步：三维度数据交叉验证
第五步：反偏见与 rug pull 检查
第六步：决策框架与输出（含监控清单）
```

**必须首先确认**：

## 第一步：安全审计（前置强制步骤）

**⚠️ 任何交易前必须先执行**

```bash
python3 scripts/bitget_api.py security --chain <chain> --contract <address>
```

**关键风险指标**：

| 字段 | 危险信号 | 行动 |
|------|---------|------|
| `highRisk = true` | 严重安全问题 | **禁止交易，直接 PASS** |
| `cannotSellAll = true` | 无法卖出全部持仓 | **重大红旗，禁止交易** |
| `isHoneypot = true` | 蜜罐合约 | **禁止交易** |
| `isProxy = true` | 可升级合约 | 提示风险，谨慎对待 |
| `buyTax / sellTax > 5%` | 高交易税 | 计入成本，评估合理性 |
| `isBlacklist = true` | 存在黑名单功能 | 重大风险，谨慎对待 |
| `isMintable = true` | 可增发 | 评估团队可信度 |
| `ownerPercentage > 10%` | 团队持仓过高 | 评估砸盘风险 |

## 第二步：16大分析模块

### 模块A：代币经济学 (Tokenomics)

**核心问题**：代币设计是否合理？价值捕获机制是什么？

**必须回答**：
- **A1. 总供应量与流通量**：
  - 总供应量 (Total Supply)
  - 流通供应量 (Circulating Supply)
  - 流通率 = 流通量 / 总量
  - 解锁时间表（是否有大量代币即将解锁？）

- **A2. 分配结构**：
  - 团队/基金会持仓比例
  - 投资者/机构持仓比例
  - 社区/空投分配比例
  - 流动性池占比

- **A3. 价值捕获机制**：
  - 代币用途（治理、质押、手续费折扣、收益分享）
  - 收入是否回流到代币（回购、销毁）
  - 质押收益率（APY）

**数据来源**：Bitget token-info API

---

### 模块B：市场数据与价格行为

**核心问题**：价格走势如何？流动性是否充足？

**必须回答**：
- **B1. 价格数据**：
  - 当前价格
  - 24h/7d/30d 涨跌幅
  - 历史最高价/最低价 (ATH/ATL)
  - 距离 ATH 的回撤幅度

- **B2. 市值指标**：
  - 完全稀释估值 (FDV)
  - 流通市值 (Market Cap)
  - FDV/MC 比率（越高说明解锁压力越大）

- **B3. 交易量分析**：
  - 24h 交易量
  - 交易量/市值比率（换手率）
  - 交易量趋势（上升/下降）

**数据来源**：Bitget token-price, K-line API

---

### 模块C：流动性与滑点分析

**核心问题**：能否顺利进出？大额交易冲击成本如何？

**必须回答**：
- **C1. 流动性池数据**：
  - 总流动性 (TVL)
  - 主要交易对（Token/USDC, Token/WETH等）
  - 流动性分布（是否集中在某个池子？）

- **C2. 滑点测试**：
  - 小额交易滑点（0.1 ETH等值）
  - 中额交易滑点（1 ETH等值）
  - 大额交易滑点（5+ ETH等值）

- **C3. 流动性健康度**：
  - 流动性/市值比率（>20%为健康）
  - 流动性是否被锁定（LP锁仓）
  - 是否有单边流动性风险

**数据来源**：Bitget liquidity-pool API, swap-quote API

---

### 模块D：链上交易活动

**核心问题**：谁在买？谁在卖？交易行为健康吗？

**必须回答**：
- **D1. 交易统计**：
  - 5m/1h/4h/24h 买入量 vs 卖出量
  - 买家数量 vs 卖家数量
  - 大单交易数量（鲸鱼活动）

- **D2. 买卖压力**：
  - 净买入压力（买入 - 卖出）
  - 买卖比率趋势
  - 异常交易检测（突然放量）

- **D3. 新资金流入**：
  - 新持有者增长
  - 活跃地址数变化

**数据来源**：Bitget tx-info API

### 常见 rug pull 红旗

| 红旗 | 说明 | 行动 |
|------|------|------|
| 合约未开源 | 无法审计 | 禁止交易 |
| 流动性未锁定 | 可随时撤池 | 禁止交易 |

| 营销钱包过大 | >10% 供应 | 砸盘风险 |
| 交易税可变 | 可突然提高至99% | 禁止交易 |
| 白名单/预售过多 | 开盘抛压 | 谨慎对待 |
| 社交媒体刷量 | 假热度 | 谨慎对待 |
| 抄袭项目 | 无原创性 | 禁止交易 |
