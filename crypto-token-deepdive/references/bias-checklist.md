# 加密货币反偏见与 rug pull 检查清单

本文件是加密货币投资的"安全网"。

---

## 第一部分：认知偏见自检

### 陷阱1：FOMO（害怕错过）

**什么意思**：看到别人赚钱，怕错过机会，冲动入场。

**自检**：
- [ ] 我是否因为价格快速上涨而急于买入？
- [ ] 我是否因为"别人都在买"而买入？
- [ ] 我是否跳过了安全审计因为"来不及了"？

**强制执行**：任何交易前必须完成安全审计，FOMO 不是跳过安全检查的理由。

---

### 陷阱2：确认偏误

**什么意思**：只关注支持自己看涨的信息，忽略风险信号。

**自检**：
- [ ] 我是否主动搜索了看空这个项目的分析？
- [ ] 我是否忽略了合约风险因为"看起来没问题"？
- [ ] 我是否因为已经买入而拒绝承认错误？

---

### 陷阱3：锚定效应

**什么意思**：被历史价格锚定，认为"从高点跌了80%很便宜"。

**自检**：
- [ ] 我是否因为"从ATH跌了90%"而认为便宜？
- [ ] 我是否忽略了代币可能归零？

---

### 陷阱4：幸存者偏差

**什么意思**：只看到成功的 meme 币，忽略了成千上万的归零项目。

**自检**：
- [ ] 我是否因为"SHIB涨了1000倍"而认为这个也会？
- [ ] 我是否了解这个赛道的真实成功率？

---

### 陷阱5：过度自信

**什么意思**：认为自己能精准抄底逃顶。

**自检**：
- [ ] 我是否设置了止损？
- [ ] 我是否投入了无法承受损失的资金？
- [ ] 我是否因为上次赚钱而加大仓位？

## 第二部分：Rug Pull 红旗检查

### 🚨 绝对禁止交易的红旗

| 红旗 | 检查方法 | 后果 |
|------|---------|------|
| 合约未开源 | 区块链浏览器查看 | 无法审计，禁止交易 |
| 流动性未锁定 | 查看 LP 锁仓状态 | 可随时撤池，禁止交易 |
| 无法卖出全部 | security API 检查 | 蜜罐合约，禁止交易 |
| 交易税可变 | 合约代码检查 | 可突然提高至99%，禁止交易 |
| 黑名单功能 | security API 检查 | 可禁止特定地址交易，禁止交易 |
| 可无限增发 | security API 检查 | 通胀风险极高，禁止交易 |
| 合约有自毁功能 | 合约代码检查 | 可随时销毁，禁止交易 |

### ⚠️ 高风险黄旗

| 黄旗 | 风险说明 | 行动 |
|------|---------|------|
| 团队完全匿名 | 无法追责 | 极高风险，极小仓位 |
| 营销钱包 >10% | 砸盘风险 | 监控钱包动向 |
| 预售/白名单过多 | 开盘抛压 | 等开盘稳定后再考虑 |
| 社交媒体刷量 | 假热度 | 用工具检测真实互动 |
| 抄袭其他项目 | 无原创性 | 谨慎对待 |
| 合约有代理模式 | 可升级 | 评估团队可信度 |
| 流动性 < $100k | 滑点极高 | 无法大额交易 |
| 持有者 < 100 | 极度集中 | 操纵风险 |

## 第三部分：Pre-Mortem（事前尸检）

假设你买入后亏了 50%，倒推原因：

### 失败路径A：合约风险
- **场景**：项目方突然提高交易税至99%，或撤走流动性
- **早期信号**：合约有后门、团队匿名、流动性未锁定
- **Kill Condition**：发现合约风险立即退出

### 失败路径B：市场情绪逆转
- **场景**：大盘暴跌，山寨币血流成河
- **早期信号**：BTC 跌破关键支撑、资金费率转负
- **Kill Condition**：BTC 趋势逆转减仓

### 失败路径C：项目方跑路
- **场景**：团队停止开发，社交媒体沉默
- **早期信号**：GitHub 无更新、社区无人回应
- **Kill Condition**：开发停滞超过1个月退出

### 失败路径D：被竞品超越
- **场景**：竞品推出更好产品，用户流失
- **早期信号**：TVL 持续下降、竞品 TVL 上升
- **Kill Condition**：失去赛道龙头地位退出

## 第四部分：仓位管理铁律

1. **单币仓位 ≤ 投资组合 10%**（高信心项目 ≤ 20%）
2. **新币/土狗仓位 ≤ 投资组合 2%**
3. **必须设止损**：-20% 或 -30% 强制退出
4. **必须设止盈**：达到目标价位分批卖出
5. **不 all in**：永远保留现金应对黑天鹅

## 第五部分：检查清单总结

交易前必须完成：

- [ ] 安全审计通过（无 highRisk）
- [ ] 流动性锁定确认
- [ ] 合约已开源
- [ ] 团队背景调查
- [ ] 代币经济学审查
- [ ] 链上数据验证
- [ ] 新闻情绪检查
- [ ] 社交媒体真实性验证
- [ ] 止损价位设定
- [ ] 仓位比例确认

全部通过才能交易。任何一项不通过 = NO TRADE。
