## 1. 巴菲特视角：Owner Earnings 估值法

### 思想来源

巴菲特在1986年致股东信中提出了"所有者收益"（Owner Earnings）的概念。他认为，传统的会计利润并不能反映企业主真正能拿走的钱。

### 核心公式

```
Owner Earnings = 净利润
              + 折旧与摊销（D&A）
              - 维持性资本支出（Maintenance CapEx）
              - 营运资本变动（Working Capital Changes）
```

**通俗理解**：净利润是账上赚了多少，但还要考虑设备折旧不用真花钱（加回来），设备坏了要换（减掉维修费），进了一堆原料还没用（钱被占用了）。剩下的才是真的能拿回家的钱。

### 计算步骤

1. **获取数据**：
   - 净利润（Net Income）：从利润表获取
   - 折旧与摊销（D&A）：现金流量表
   - 总资本支出（Total CapEx）：现金流量表"investing activities"
   - 维持性CapEx估算：通常取总CapEx的40-60%（扩张期科技公司可能只占20-30%）

2. **计算 Owner Earnings**

3. **用10-15倍 Owner Earnings 估算"巴菲特合理价格"**：
   - 高确定性业务（强护城河成熟科技公司）：12-15倍
   - 一般确定性：10-12倍

### 判断标准

| Owner Earnings 倍数 | 判断 |
|-------------------|------|
| 当前股价 < 10x OE | 明显低估（按巴菲特标准） |
| 当前股价 = 10-15x OE | 合理区间 |
| 当前股价 = 15-25x OE | 偏贵，需要高增长支撑 |
| 当前股价 > 25x OE | 高估（除非增长极为确定） |

## 2. 彼得·林奇视角：PEG Ratio 估值法

### 思想来源

彼得·林奇（Peter Lynch），麦哲伦基金传奇经理人，创造了13年29.2%年化收益率的纪录。

### 核心公式

```
PEG Ratio = 市盈率（P/E） ÷ 盈利增长率（EPS Growth Rate %）
```

**通俗理解**：如果一家公司市盈率是30倍，但利润每年增长30%，那PEG=1，说明付出的价格和增长速度匹配。如果PEG=2，说明为每单位增长付了双倍的钱——太贵了。

### 计算步骤

1. 获取远期PE（Forward P/E）：基于未来12个月预期盈利
2. 获取盈利增长率：分析师共识预期的未来3-5年年均EPS增长率
3. 计算 PEG

### 判断标准

| PEG Ratio | 判断 |
|-----------|------|
| < 0.5 | 极具吸引力（可能被严重低估） |
| 0.5 - 1.0 | 有吸引力（增长与估值匹配良好） |
| 1.0 - 1.5 | 合理（对高质量科技公司可接受） |
| 1.5 - 2.0 | 偏贵（需要超强竞争优势支撑） |
| > 2.0 | 昂贵（除非有极特殊的增长催化剂） |

## 3. 反向DCF视角：Reverse DCF

### 思想来源

传统DCF是给一大堆未来假设然后算出"内在价值"。问题是：垃圾假设进去，垃圾估值出来（Garbage In, Garbage Out）。反向DCF的逻辑完全反过来——从当前股价出发，反推出"市场正在定价的隐含增长率是多少"。

### 核心逻辑

```
当前股价 → 反推 → 市场隐含的未来收入/盈利增长率
↓
你来判断：这个隐含增长率，合理吗？
```

### 简化计算步骤

1. **获取数据**：
   - 当前市值（Market Cap）或企业价值（EV = 市值 + 净负债）
   - 当前自由现金流（FCF）或营业利润
   - WACC：科技股通常用8-12%。大型科技8-10%，中型科技10-12%
   - 终值增长率：通常假设2-3%（长期GDP增速附近）

2. **反推隐含增长率**：
   ```
   当前EV = FCF × (1+g) / (WACC-g) × [增长期乘数]
   解方程求 g（隐含增长率）
   ```

3. **判断隐含增长率是否合理**：
   - 与公司历史增长率对比
   - 与行业增长率对比
   - 与管理层指引对比
   - 与分析师预期对比

### 判断标准

| 隐含增长率 vs 合理预期 | 判断 |
|---------------------|------|
| 隐含增长率 < 历史增速 & 分析师预期 | 可能被低估（市场悲观过度） |
| 隐含增长率 ≈ 合理预期 | 合理定价 |
| 隐含增长率 > 预期20%以上 | 定价偏乐观，需要"超预期"表现 |
| 隐含增长率远超行业天花板 | 严重高估，市场在做白日梦 |

## 4. 格林布拉特视角：魔法公式（Magic Formula）

### 思想来源

乔尔·格林布拉特（Joel Greenblatt），《股市稳赚》作者。

### 核心公式

```
指标1：盈利收益率（Earnings Yield）= EBIT / 企业价值（EV）
→ 反映"便宜程度"

指标2：投入资本回报率（ROIC）= EBIT / (净营运资本 + 净固定资产)
→ 反映"质量"
```

### 判断标准

| 维度 | 优秀 | 良好 | 一般 | 差 |
|------|------|------|------|-----|
| Earnings Yield | > 8% | 5-8% | 3-5% | < 3% |
| ROIC | > 25% | 15-25% | 10-15% | < 10% |

**组合判断**：
- Earnings Yield 高 + ROIC 高 = ⭐ 魔法公式首选（又好又便宜）
- Earnings Yield 低 + ROIC 高 = 好公司但贵了（等回调）
- Earnings Yield 高 + ROIC 低 = 便宜但质量一般（可能是价值陷阱）
- Earnings Yield 低 + ROIC 低 = 两个维度都不吸引

## 5. 达摩达兰视角：EV/EBITDA 行业对标

### 思想来源

阿斯瓦斯·达摩达兰（Aswath Damodaran），纽约大学教授，"估值教父"。

### 核心指标

```
EV/EBITDA = 企业价值 ÷ 息税折旧摊销前利润
```

**为什么用EV/EBITDA而不是P/E？**
- EV考虑了负债，比单纯市值更全面
- EBITDA剔除了资本结构、税率差异、折旧政策差异的影响

### 科技子行业参考基准

| 子行业 | 典型EV/EBITDA范围 | 说明 |
|--------|------------------|------|
| 大型SaaS/云 | 20-40x | 高增长、高经常性收入溢价 |
| 成熟软件 | 15-25x | 稳定现金流 |
| 半导体 | 12-20x | 周期性折价 |
| 消费互联网 | 15-30x | 取决于增长和货币化能力 |
| 硬件/设备 | 8-15x | 较低倍数 |

### 判断标准

| 相对行业中位数 | 判断 |
|-------------|------|
| 低于行业中位数20%+ | 可能被低估（需排除基本面恶化原因） |
| 在行业中位数附近（±10%） | 合理定价 |
| 高于行业中位数20-50% | 溢价定价（需要更优的增长或质量支撑） |
| 高于行业中位数50%+ | 高溢价（需要非常强的理由） |

## 6. 科技股专属视角：EV/Revenue + Rule of 40

### 核心指标

```
指标1：EV/Revenue（企业价值 / 收入）
→ 每1块钱收入被市场定价多少钱

指标2：Rule of 40 分数 = 收入增长率(%) + 利润率(%)
→ 衡量"增长 vs 盈利"的平衡
```

### EV/Revenue 判断基准

| EV/Revenue | 增长率 > 30% | 增长率 15-30% | 增长率 < 15% |
|-----------|-------------|-------------|-------------|
| < 5x | 极具吸引力 | 有吸引力 | 可能有原因的便宜 |
| 5-10x | 有吸引力 | 合理 | 偏贵 |
| 10-20x | 合理 | 偏贵 | 昂贵 |
| 20-40x | 可接受（高增长溢价） | 昂贵 | 极度高估 |
| > 40x | 需要极强叙事 | 泡沫风险 | 泡沫 |

### Rule of 40 判断标准

| 分数 | 判断 |
|------|------|
| > 60 | 🏆 顶级——增长和盈利双优 |
| 40-60 | ✅ 健康——平衡良好 |
| 25-40 | 🟡 及格线以下——需要在某个维度改善 |
| < 25 | ❌ 不及格——增长和盈利都有问题 |

## 敏感性分析（必须执行）

**为什么必须做**：单点估值（"合理价格是$250"）创造了虚假的精确感。敏感性分析展示"如果我的假设偏差了10-20%，估值会怎么变"。

**执行方法**：为每个主要估值模型选择2个最关键的输入变量，构建二维矩阵。

## Action Price 推导（独立于当前股价）

**关键理念**：如果你的"买入价"是通过"当前股价打八折"得来的，你被锚定了。

正确做法：
1. **完全不看当前股价**，独立运行你的估值模型
2. 得出一个"合理价值区间"
3. 在合理价值的基础上，要求15-20%的安全边际（Margin of Safety）
4. 安全边际之后的价格 = 你的 Action Price
5. **然后**才去看当前股价，比较Action Price和市场价

```
独立估值 → 合理价值 → 减去安全边际 → Action Price → 对比当前股价
```

## IRR 门槛检查（回报率门槛）

**IRR（Internal Rate of Return）**：这笔投资的预期年化回报率。

```
IRR ≈ (目标价 / 当前价格)^(1/年数) - 1
```

| 仓位类型 | 最低IRR门槛 | 理由 |
|---------|-----------|------|
| 做多（核心持仓） | ≥ 15% | 必须大幅跑赢指数才值得承担集中风险 |
| 做多（战术持仓） | ≥ 20% | 短期持仓机会成本更高 |
| 做空 | ≥ 20-25% | 补偿做空的无限下行风险、借券成本和时机风险 |

**铁律**："好公司+合理价格" = PASS，不是BUY。"好公司+有吸引力的价格+IRR超过门槛" = BUY。

如果计算出的预期IRR未达门槛，无论公司基本面多好、叙事多动人，推荐必须是PASS + 注明"在$X价格我们会行动"。
