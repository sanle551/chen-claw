# 反偏见与盲区检查清单

本文件是投资分析的"安全网"。在完成财报拆解和估值模型后，必须逐条过一遍这个清单。

---

## 第一部分：认知偏见自检（6大陷阱）

### 陷阱1：确认偏误（Confirmation Bias）

**什么意思**：你已经有了一个看法，然后只关注支持这个看法的信息，忽略或轻视反面证据。

**自检问题**：
- [ ] 我是否主动搜索了看空这家公司的分析文章？
- [ ] 如果有分析师给出"卖出"评级，我是否认真读了他们的理由？
- [ ] 我的分析中，正面论点和负面论点的篇幅和深度是否平衡？
- [ ] 我是否因为已经持有这只股票，而倾向于看好它？

**强制执行**：搜索 `[股票代码] bear case` 或 `[股票代码] risks concerns sell`，找到至少2条最强的反对意见，并在报告中完整呈现。

---

### 陷阱2：锚定效应（Anchoring Bias）

**什么意思**：你的判断被"锚"拉住了——通常是历史股价、你的买入成本、或者曾经的最高价。

**自检问题**：
- [ ] 我是否在不自觉地参考历史最高价或52周高/低价来判断"便宜"或"贵"？
- [ ] 如果我今天第一次看到这家公司（完全不知道它的历史价格），我愿意以当前价格买入吗？
- [ ] 我的估值模型的核心参数是基于独立分析，还是被"历史大概是这样"锚住了？
- [ ] 我是否因为自己的买入成本而影响了持有/卖出的判断？

**强制执行**：在估值环节，尝试不看当前股价，先独立计算出一个合理估值区间，然后再与当前股价比较。

---

### 陷阱3：叙事谬误（Narrative Fallacy）

**什么意思**：人类大脑天生爱听故事。一个好的投资"故事"会让你对数字的要求变低。好故事 ≠ 好投资。

**自检问题**：
- [ ] 如果我把这家公司的"故事"完全抽掉，只看财务数字，我还会同样看好吗？
- [ ] 管理层讲的故事是否有充分的数据支撑，还是更多是愿景和承诺？
- [ ] 这个叙事是否有一个可以被证伪的时间节点？

**强制执行**：在报告中明确写出"如果剥离叙事，纯看财务数据的结论是什么"。

---

### 陷阱4：从众心理 / 权威偏见（Herding / Authority Bias）

**什么意思**：因为"所有人都在买"或"某个著名投资人买了"就跟着买。放弃了自己的独立判断。

**自检问题**：
- [ ] 我的看法是基于自己的分析，还是主要因为看到了别人的持仓或推荐？
- [ ] 如果一个著名基金经理突然清仓了这只股票，我会改变看法吗？
- [ ] 当前的市场共识是什么？我的观点与共识一致还是有差异？
- [ ] 这只股票的分析师评级分布如何？是否已经一边倒地看好？

**强制执行**：搜索 `[股票代码] analyst ratings distribution` 和 `[股票代码] hedge fund holdings`，了解当前市场共识的方向和强度。

---

### 陷阱5：处置效应（Disposition Effect）

**什么意思**：投资者倾向于"过早卖出赢家，过晚卖出输家"。赚了一点就想落袋为安，亏了很多却死扛不卖。

**自检问题**（如果已经持有）：
- [ ] 如果我今天空仓，看到这家公司的全部信息，我会选择以当前价格买入吗？
- [ ] 我继续持有的理由是基于未来前景，还是因为"已经亏了不甘心卖"？
- [ ] 我的"止损线"或"止盈线"是在买入前就设定的，还是在不断调整？

**强制执行**：在报告的操作建议部分，假设分析者是从零开始评估这家公司，而不是已经有持仓。

---

### 陷阱6：过度自信（Overconfidence Bias）

**什么意思**：高估自己预测未来的能力、低估不确定性、仓位过于集中。

**自检问题**：
- [ ] 我的估值假设中，有没有使用单一点估计而非区间？
- [ ] 如果我的核心假设有20%的误差，估值会变成什么样？
- [ ] 我是否给了这家公司在投资组合中过高的权重？（单只股票占比超过15-20%就需要非常强的理由）

**强制执行**：估值必须给出区间（低/中/高情景），而不是单一数字。在操作建议中明确仓位上限。

## 第二部分：财务红旗检查（7大警示信号）

### 红旗1：收入确认异常

**检查**：
- [ ] 是否有会计政策变更影响收入确认时点？
- [ ] 递延收入（Deferred Revenue）是在增长还是萎缩？（对SaaS公司尤其重要）
- [ ] 是否有大额一次性收入拉高了本季数字？

### 红旗2：GAAP vs Non-GAAP 差异过大

**检查**：
- [ ] Non-GAAP利润比GAAP利润高出多少？如果差距超过50%，需要仔细看差异来源
- [ ] 股票期权费用（SBC）占收入的比例是多少？超过15-20%意味着大量稀释
- [ ] 是否有频繁的"一次性"调整项？

### 红旗3：应收账款增速 > 收入增速

**检查**：
- [ ] 应收账款的增长率是否大幅超过收入增长率？
- [ ] DSO（Days Sales Outstanding，应收账款周转天数）是否在上升？

### 红旗4：管理层行为异常

**检查**：
- [ ] 高管近期是否有大量抛售自家股票？
- [ ] 是否有关键高管离职？特别是CFO
- [ ] 审计师是否更换？无合理理由的审计师变更是最严重的红旗之一

### 红旗5：资本支出暴增但回报不明

**检查**：
- [ ] 资本支出是否大幅增加？增加的钱花在了哪里？
- [ ] 历史上的大额CapEx是否产生了可衡量的回报？
- [ ] CapEx占收入的比例是否远高于同行？

### 红旗6：现金流与利润长期背离

**检查**：
- [ ] 是否连续多个季度出现"利润增长、现金流下降"？
- [ ] 经营活动现金流是否为负？

### 红旗7：负债结构恶化

**检查**：
- [ ] 近期是否有大额债务到期（refinancing wall）？
- [ ] 信用评级是否被下调或有被下调的风险？
- [ ] 利息覆盖倍数（EBIT / 利息支出）是否低于3倍？

## 第三部分：科技股特有的盲区（5大盲区）

### 盲区1：TAM幻觉（Total Addressable Market Illusion）

科技公司喜欢宣称自己面对的潜在市场有几千亿美元。但TAM不等于公司真的能拿到的市场份额。

**检查**：管理层声称的TAM是否合理？公司实际可触达的SAM（Serviceable Addressable Market）是多少？

### 盲区2：AI收入的真实性

当前周期中，几乎每家科技公司都在强调自己的"AI收入"。但很多所谓的"AI收入"可能是：重新贴标签的已有产品收入、一次性的硬件采购、或者客户还在"试水"阶段。

**检查**：
- AI相关收入的具体定义是什么？
- 这些AI收入是经常性的还是一次性的？
- 客户是在试用阶段还是已经大规模部署？

### 盲区3：股票期权稀释的隐蔽成本

科技公司大量使用股票期权（Stock-Based Compensation）来吸引人才。这不是现金支出，但它在不断稀释现有股东的权益。

**检查**：
- SBC占收入的百分比是多少？年度趋势如何？
- 流通股数量（Shares Outstanding）每年增长多少？
- 计算"真实每股价值增长" = 总利润增长率 - 股权稀释率

### 盲区4：客户获取成本的拐点

SaaS和消费互联网公司在早期增长阶段，往往能以较低成本获取大量用户。但随着渗透率提高，获取新客户的边际成本会急剧上升。

**检查**：
- CAC（Customer Acquisition Cost）的趋势是上升还是下降？
- LTV/CAC比率是否在恶化？
- 如果削减营销，增长还能维持吗？

### 盲区5：监管尾部风险

科技巨头面临的监管风险往往被市场长期低估，直到监管真的落地时才反映在股价里。

**检查**：
- 是否有正在进行的反垄断调查或诉讼？
- AI监管政策的潜在影响
- 数据隐私法规的合规成本
- 地缘政治风险（尤其是中美关系对有大量中国收入的科技公司的影响）

---

## 使用指南

完成检查后，将结果分为三类纳入报告：

1. **🔴 红旗警报**：发现了明确的风险信号，必须在投资决策中予以重大考量
2. **🟡 黄旗提醒**：存在潜在担忧，需要持续监控，可能影响持仓时长或仓位大小
3. **✅ 绿灯确认**：经检查后确认的积极因素

## 第四部分：Pre-Mortem 事前尸检

这是整个反偏见框架中最具穿透力的工具。常规风险分析是"可能有什么风险"，Pre-Mortem是"**假设已经亏了大钱，倒推是怎么亏的**"。

### 执行方法

想象时间是2年后。你持有这只股票，亏了40%以上。

**强制回答3个问题**：

**问题1：最可能的失败路径有哪些？**

给出2-3条独立的失败路径，每条包括：
- 发生了什么（具体场景描述）
- 为什么会发生（驱动因素）
- 今天能看到什么早期预警信号

**问题2：我今天在做什么假设？哪些假设如果错了，后果最严重？**

列出你最核心的3个假设，以及每个假设被证伪的概率和后果。

**问题3：如果一个非常聪明的人在另一面（和你持相反的仓位），他们看到了什么你没看到的？**

搜索 `[代码] short thesis` `[代码] bear case strongest argument` 来找到最聪明的反对者的论点。

### Pre-Mortem输出格式

```
## Pre-Mortem：如果2年后亏了钱

### 失败路径A：[标题]
- 场景：[具体描述]
- 驱动因素：[为什么会发生]
- 概率评估：[低/中/高]
- 今日早期信号：[现在能观察到什么]
- 对应Kill Condition：[什么阈值触发退出]

### 失败路径B：[标题]
[同上格式]

### 失败路径C：[标题]
[同上格式]

### 核心假设脆弱性
| 假设 | 如果错了的后果 | 被证伪的可能性 |
|------|-------------|-------------|
| [假设1] | [后果] | [低/中/高] |
| [假设2] | [后果] | [低/中/高] |
| [假设3] | [后果] | [低/中/高] |
```
